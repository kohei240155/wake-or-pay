# WakeOrPay 手動 E2E 検証ガイド

## 前提条件

- Node.js 20+
- Vercel アカウント
- Upstash QStash アカウント（トークン・署名キー取得済み）
- OpenAI API キー

## 1. ローカルセットアップ

```bash
cp .env.local.example .env.local
# .env.local を編集して実際の値を設定
npm install
npm run dev
```

## 2. Vercel デプロイ

```bash
npx vercel
```

Vercel ダッシュボードで環境変数を設定:

| 変数名 | 説明 |
|---|---|
| `OPENAI_API_KEY` | OpenAI API キー |
| `QSTASH_TOKEN` | QStash トークン |
| `QSTASH_CURRENT_SIGNING_KEY` | QStash 現在の署名キー |
| `QSTASH_NEXT_SIGNING_KEY` | QStash 次の署名キー |
| `APP_BASE_URL` | デプロイ先の URL（例: `https://wake-or-pay.vercel.app`） |

## 3. 検証シナリオ

### シナリオ A: 正常解除

1. ブラウザでアプリを開く
2. 2分後の時刻をセットする
3. カウントダウンが表示されることを確認
4. 「解除」ボタンを押す
5. **期待結果**: 「解除完了！ペナルティなし。いい朝ですね。」が表示される

### シナリオ B: ペナルティ発火

1. ブラウザでアプリを開く
2. 2分後の時刻をセットする
3. 解除せずに時刻が経過するまで待つ
4. **期待結果**: Vercel ログに `[punish] alarmId=<id> status=success` が出力される
5. ブラウザを再読み込みすると「ペナルティ執行済」が表示される

### シナリオ C: 過去時刻バリデーション

1. ブラウザでアプリを開く
2. 過去の時刻を入力して「セット」を押す
3. **期待結果**: 「過去の時刻は設定できません」エラーが表示される

### シナリオ D: 解除失敗リトライ

1. アラームをセットする
2. ネットワークを切断してから「解除」を押す
3. **期待結果**: エラーメッセージが表示される
4. ネットワークを再接続して「解除」を再度押す
5. **期待結果**: 解除が成功する
